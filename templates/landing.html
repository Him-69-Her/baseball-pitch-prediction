// Protocol

How a kilowatt moves.

01
Match

PostGIS proximity matching pairs solar surplus with nearby demand. Line-loss physics scores every possible trade in real time. Closer wins. The market is spatial, not abstract.

02
Settle

Trades batch-aggregate off-chain, then settle on Arbitrum L2 through an optimistic pipeline. ~95% gas savings vs per-trade on-chain. ERC-4337 account abstraction handles the UX.

03
Deliver

Electrons flow over the existing ComEd / Ameren wire. Green Button meter data reconciles the ledger. FERC Order 2222 provides the federal preemption shield over state classification.

// Thesis

Not a solar app.
A matching engine.

Tiny-Hub is a deterministic spatial matching engine. The core is three things: a PostGIS proximity graph, real-time grid physics for line-loss cost, and optimistic batch settlement on L2.

The same architecture extends to orbital mechanics with parameter substitutions — not rewrites. Space-based solar power downlinks. Orbital data-center load. Delay-tolerant settlement for deep-space relays. Phase 8 (Horizon) is roadmapped.

Today it runs across D63 (ComEd / PJM) and D91 (Ameren / MISO) in Illinois. Pilot deployment in Woodstock, IL.

The raise: $2M seed. I built the Ferrari. This funds the pit crew.
